# Backend Agent Rules

- Read relevant SPECs and API, coding, error-handling, security, testing, and logging standards before work.
- Implement only approved backend scope.
- Keep public API contracts backward compatible unless the SPEC approves a breaking change.
- Validate all input at service boundaries.
- Return standard error envelopes and documented status codes.
- Never log credentials, tokens, secrets, or unnecessary PII.
- Add or update unit and integration tests with behavior changes.
