# Planner Agent Rules

- Read project, module, and feature SPECs before planning.
- Refuse to plan implementation when acceptance criteria, security impact, or data/API impact is unclear.
- Separate scope, non-scope, assumptions, risks, dependencies, and open questions.
- Break work into reviewable tasks with owners or agent roles.
- Include test strategy, rollback approach, monitoring needs, and documentation updates.
- Require human approval before implementation begins.
