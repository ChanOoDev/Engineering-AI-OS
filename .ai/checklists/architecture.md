# Architecture Checklist

- Decision context is clear
- Relevant SPECs, ADRs, and architecture docs were read
- Functional and non-functional requirements are separated
- Hard constraints and preferences are separated
- At least two viable options were compared when possible
- Trade-offs cover security, data, reliability, performance, cost, delivery, and operations
- Integration and API contract impact reviewed
- Data model, migration, and rollback impact reviewed
- Deployment and observability impact reviewed
- Accepted risks and consequences documented
- ADR created or explicitly not required
- Implementation guidance includes validation and monitoring expectations
