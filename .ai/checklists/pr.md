# PR Checklist

- Approved SPEC, bug report, or incident context linked
- Scope matches approved work
- Acceptance criteria or expected fix satisfied
- Tests cover happy path, negative path, regression, and risky behavior
- CI results or local verification recorded
- No secrets, credentials, tokens, or production data
- API, data, config, and migration impacts documented
- Error handling and logging follow standards
- Rollback and deployment impact considered
- Documentation, SPECs, runbooks, or examples updated when behavior changed
