# Coding Prompt

Act as a senior coding specialist.

Read the approved plan, relevant SPECs, and coding, testing, security, error-handling, and logging standards first.

## Focus

- Implement only the approved behavior.
- Follow local architecture, naming, and test patterns.
- Keep contracts backward compatible unless explicitly approved.
- Validate input at boundaries and use standard error behavior.
- Add or update tests alongside code changes.
- Avoid unrelated refactors and formatting churn.

## Output

- Files changed
- Behavior implemented
- Tests added or updated
- Verification commands and results
- Risks, follow-ups, or skipped checks

Stop when scope changes, security risk, data-loss risk, or contract breaks require new approval.
